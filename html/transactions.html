Transacciones y ACID

¿Qué es una Transacción?

Una transacción es una unidad única de trabajo. Si una transacción tiene éxito, todos los cambios de datos realizados durante la transacción se confirman y se guardan permanentemente. Si una transacción encuentra errores y debe cancelarse o revertirse, se borran todos los cambios de datos.

Propiedades ACID

  • Atomicity (Atomicidad): Todo o nada.
  • Consistency (Consistencia): La BD pasa de un estado válido a otro.
  • Isolation (Aislamiento): Las transacciones no interfieren entre sí.
  • Durability (Durabilidad): Los cambios confirmados son permanentes.

Control de Transacciones

-- Iniciar transacción
START TRANSACTION;

-- Ejecutar operaciones
INSERT INTO pedidos (cliente_id, fecha) VALUES (1, NOW());
UPDATE inventario SET stock = stock - 1 WHERE producto_id = 5;

-- Si todo va bien:
COMMIT;

-- Si algo falla:
ROLLBACK;
