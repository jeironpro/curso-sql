Seguridad y Permisos

Gestión de Usuarios

Es fundamental controlar quién tiene acceso a la base de datos y qué puede hacer.

Crear Usuario

CREATE USER 'nuevo_usuario'@'localhost' IDENTIFIED BY 'contraseña_segura';

Permisos (GRANT y REVOKE)

Dar Permisos (GRANT)

-- Dar permiso de lectura en todas las tablas
GRANT SELECT ON mi_base_datos.* TO 'nuevo_usuario'@'localhost';

-- Dar todos los permisos
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost';

Quitar Permisos (REVOKE)

REVOKE INSERT, UPDATE ON mi_base_datos.* FROM 'nuevo_usuario'@'localhost';

Aplicar Cambios

FLUSH PRIVILEGES;
